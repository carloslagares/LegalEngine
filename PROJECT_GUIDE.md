# LegalEngine — Technical Project Guide

## Overview

LegalEngine is a **Chekin-branded compliance hub** that helps vacation rental owners and property managers understand and navigate the legal requirements for registering their properties across Spain and Europe. It combines a real-time AI chatbot powered by Claude, downloadable PDF compliance guides, lead capture for legal services, an admin panel for managing guides and leads, and an analytics dashboard.

**Live URL:** `http://localhost:3000` (Node.js + Express)
**Admin Panel:** `http://localhost:3000/admin`

## Architecture

### Frontend (`index.html`)

A single-page application with no build step — pure HTML, CSS, and vanilla JavaScript. Key sections:

| Section | Description |
|---------|-------------|
| **Header** | Chekin-branded mega-menu navigation (Features, Solutions, Resources, Pricing, Countries) with dropdowns, language selector (EN/ES), and CTAs |
| **Hero** | Search bar with typewriter placeholder animation. Transitions to chat mode on first question |
| **Chat Interface** | WhatsApp-style chat with SSE streaming, follow-up suggestions, "New topic" button |
| **Region Cards** | Dynamic grid loaded from `/api/regions`, grouped by country (Spain, Italy) with emoji icons |
| **Lawyer CTA** | Lead capture form for connecting users with specialized lawyers |
| **FAQ** | Accordion-style frequently asked questions |

**Key Frontend Features:**
- **SSE Streaming**: Real-time token-by-token rendering from Claude API
- **Region Detection**: Client-side `detectRegionSlug()` maps 50+ city/province names to guide slugs
- **Thematic Detection**: `isThematicQuestion()` identifies topic-based questions (invoicing, taxes, insurance, etc.) to skip the region picker
- **PDF Generation**: Uses jsPDF (v2.5.1) to draw PDFs directly — cover page, step pages, checklist — no html2canvas dependency
- **Bilingual Guides**: PDF content available in EN/ES, selected based on `data-lang` attribute
- **Lead Capture**: Two modal forms — one for lawyer referrals, one for guide downloads — with property type toggles and country/region pre-fill
- **Dynamic Region Cards**: Loaded from `/api/regions` at startup, grouped by country with emoji icons and descriptions
- **Chat Session Tracking**: Each chat session generates a UUID (`crypto.randomUUID()`) sent with every message for server-side persistence. Resets on "New topic"

### Backend (`server.js`)

Express server handling static files, AI chat, admin panel, and data management.

## AI Integration

### Model & SDK

- **Model**: Claude Sonnet (`claude-sonnet-4-20250514` by default, configurable via `CLAUDE_MODEL` env var)
- **SDK**: `@anthropic-ai/sdk` (official Anthropic Node.js SDK)
- **Streaming**: Server-Sent Events (SSE) for real-time response rendering

### Request Flow

```
User question + sessionId
     |
     v
Cache check (MD5 hash of question+region+lang)
     |
     +-- Cache HIT --> Return cached response instantly (no API call)
     |
     +-- Cache MISS
            |
            v
      RAG enabled? (VOYAGE_API_KEY set)
            |
            +-- YES --> resolveGuidesRAG() via Voyage AI embeddings
            |              - Embed question with voyage-3-lite
            |              - Cosine similarity against chunk index
            |              - Return top chunks (score > 0.35)
            |
            +-- NO --> resolveGuides() keyword matching
                          - Check region parameter
                          - Scan aliases + CITY_TO_REGION map
                          - Check THEMATIC_TRIGGERS regex
                          - Check EUROPEAN_LOCATIONS array
            |
            v
      Build messages array with guide context
            |
            v
      Claude API (streaming) + System prompt
            |
            v
      SSE chunks --> Frontend renders token-by-token
            |
            v
      Post-processing:
        - Store chat session (upsert by sessionId)
        - Cache response (24h TTL, first messages only)
        - Track analytics event
        - Extract <follow-up> tags --> clickable suggestion buttons
```

### System Prompt Design

The system prompt establishes Claude as the **Chekin Compliance Assistant** with these rules:

1. **Primary source**: Answer from guide content when available
2. **Supplemental knowledge**: When guides don't cover a topic, Claude can use its own knowledge (flagged as such)
3. **Language lock**: Respond in the website's current language (EN/ES). Only switch if the user explicitly writes in another language
4. **HTML formatting**: Responses use `<strong>`, `<ul>/<li>`, `<ol>/<li>`, `<p>` for rich chat rendering
5. **Follow-up generation**: Every response ends with `<follow-up><q>...</q></follow-up>` tags for suggested next questions
6. **Context continuity**: Short follow-ups (city names, "yes"/"no") are treated as topic continuations, not new queries

### Guide Context Injection

Relevant guides are prepended to the first user message as XML:

```xml
<guide region="Andalucia">
[Markdown content — full guide or RAG chunks]
</guide>

Question: How do I register my VUT in Malaga?
```

When RAG is active, only the most relevant chunks are injected instead of the full guide, reducing token usage and improving focus.

## RAG — Embedding-Based Retrieval

### How It Works

When `VOYAGE_API_KEY` is set, LegalEngine uses **Voyage AI** embeddings for semantic guide retrieval instead of keyword matching.

| Component | Detail |
|-----------|--------|
| **Model** | `voyage-3-lite` (Voyage AI) |
| **Chunking** | Guides split by headings, then paragraphs. Max 800 chars, 100 char overlap |
| **Storage** | Embeddings stored in `guide_embeddings` table (SQLite) |
| **Similarity** | Cosine similarity, threshold > 0.35, top 8 chunks |
| **Grouping** | Chunks grouped by guide slug, top 4 guides returned |
| **Caching** | Embeddings cached with content hash — only re-generated when guide content changes |
| **Fallback** | If Voyage API is unavailable or key not set, falls back to keyword matching |

### Initialization

On server startup, RAG initializes in the background (non-blocking):

1. Load existing embeddings from SQLite
2. Check content hashes against current guide content
3. For new/changed guides, chunk text and call Voyage API in batches of 64
4. Store new embeddings in SQLite for next startup
5. When a guide is uploaded/deleted via admin, the RAG index is updated automatically

## Response Caching

Reduces API costs by caching Claude responses for identical first-message questions.

| Setting | Value |
|---------|-------|
| **Scope** | First messages only (no conversation history) |
| **Key** | MD5 hash of `question + region + lang` |
| **TTL** | 24 hours (configurable via `CACHE_TTL_HOURS`) |
| **Storage** | SQLite `response_cache` table |
| **Delivery** | Cached responses served as SSE (same format as live responses) |
| **Cleanup** | Expired entries removed hourly |
| **Management** | Clear cache from admin Analytics tab |

## Admin Panel (`/admin`)

### Authentication

- Session-based auth with `express-session` + `connect-sqlite3`
- Credentials from env vars (`ADMIN_USER` / `ADMIN_PASS`), defaults: `admin` / `chekin2026`
- 24-hour session cookie (httpOnly, sameSite: lax)
- All `/api/admin/*` routes (except login) require `requireAuth` middleware

### Tabs

#### 1. Guides
- Table of all knowledge base guides (regional + thematic)
- Shows: region name with emoji, country badge, type badge, aliases, source file
- **Upload Guide**: Modal with country select (Spain/Italy) -> region select (auto-populated) -> PDF file picker
- Upload flow: receive PDF -> extract text with `pdf-parse` -> clean text -> save as `knowledge/<slug>.md` -> update `_index.json` -> reload knowledge -> re-index RAG
- **Delete Guide**: Removes `.md` file, updates index, cleans RAG embeddings
- Region metadata (name, emoji, description, aliases) auto-generated from `REGION_META` object covering all 17 Spanish CCAA + 20 Italian regions

#### 2. Leads
- Table of all captured leads with filtering: type, search (email/name/company), date range
- Pagination (50 per page)
- **Export CSV**: Download all leads as CSV file

#### 3. Chats
- Table of all chat sessions: truncated session ID, region, language, message count, timestamps
- Filters: search messages, date range
- **Click a session** to open a modal with the full conversation (user/assistant bubbles)
- Pagination (50 per page)

#### 4. Analytics
- **Stats cards**: Total questions, total leads, chat sessions, cache hit rate + RAG status
- **Charts** (powered by Chart.js):
  - Questions Over Time (30-day line chart)
  - Top Regions (horizontal bar chart)
  - Questions vs Leads conversion (dual line chart)
  - Language distribution (doughnut chart)
- **Recent Questions** table: last 20 questions with region, language, timestamp
- **Top Guide Downloads** table: download counts by region
- **Clear Cache** button

## PDF Guide Generation

### Approach

Uses **jsPDF** (v2.5.1) to draw PDFs programmatically — no HTML rendering or screen capture. Each PDF is constructed page-by-page using the jsPDF drawing API.

### PDF Structure

1. **Cover Page**: Dark navy (#0a0a2e) background, badge, title, region name in purple, intro text, version metadata, Chekin logo
2. **Step Pages** (6 per guide): Purple step number badge, title, subtitle, divider, content blocks (paragraphs, bullet lists, stat cards, subsections with letter circles)
3. **Checklist Page**: Green check badge, numbered checklist items with purple checkboxes, CTA callout

### Content Block Types

| Block Type | Rendering |
|------------|-----------|
| `p` | Paragraph text with automatic line wrapping |
| `li` | Bullet point with purple dot (supports `text` string or `items` array) |
| `h4` | Bold subheading |
| `stats` | Horizontal row of stat cards (supports `items` array or single `{label, value}`) |
| `sub` | Subsection — full card (`letter`, `title`, `desc`) or simple subheading (`text` only) |

### Bilingual Support

Guide data is structured as `guideData[slug].en` and `guideData[slug].es`. The `getSiteLang()` function reads `document.documentElement.getAttribute('data-lang')` to determine which version to render.

### Adding a New Region

**Via Admin Panel (recommended):**
1. Go to `/admin` -> Guides tab -> Upload Guide
2. Select country (Spain/Italy) and region
3. Upload the PDF file
4. The system automatically: extracts text, creates `.md` file, updates `_index.json`, reloads knowledge, re-indexes RAG, generates region metadata (emoji, description, aliases)

**Manual method:**
1. Drop the PDF into `Guias/`
2. Add mapping in `scripts/extract-guides.js`
3. Run `npm run extract`
4. Add guide content to `guideData` in `index.html` (for PDF generation)
5. The region card will appear automatically (loaded from `/api/regions`)
